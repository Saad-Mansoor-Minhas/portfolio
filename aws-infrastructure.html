An AWS stack built to survive its own failures.

A production container-hosting architecture — ECR through EC2, auto scaling, load balancing, and DNS — provisioned entirely with Terraform and deployed automatically through Bitbucket Pipelines. Designed so releases cost nothing and instance failures fix themselves.

Context

Freelance client engagement, live production workload

Stack

AWS · Terraform · Bitbucket Pipelines · Docker

Year

2023 — 2024

ECR → EC2 → Auto Scaling Group → Application Load Balancer → Route53

The brief

No ops team, no downtime budget

The client ran containerized workloads that needed to survive instance failures, absorb traffic spikes, and take new versions without downtime — and there was no dedicated ops team to babysit any of it.

The answer was a fully codified stack: Docker images flow from ECR onto EC2 instances managed by an Auto Scaling Group, fronted by an Application Load Balancer, with DNS in Route53. Every resource lives in Terraform; every deploy starts with a git push.

What I built

The work

  • Modular Terraform — reusable modules for networking, compute, and DNS, parameterized per environment.
  • Auto Scaling Group with health-check-driven instance replacement and load-based scaling policies.
  • ALB target groups with health checks for graceful, rolling instance turnover.
  • Bitbucket Pipelines building images, pushing to ECR, and rolling out via launch-template refresh.
  • Route53 alias records pointed at the ALB for clean, zone-managed DNS.
  • A parallel ECS (EC2 launch type) variant with ALB and ASG, for teams preferring managed orchestration.
Problems along the way

Challenges

Deploys dropped requests

Replacing instances naively cut off in-flight requests during every release.

Fixed with ALB connection draining plus ASG instance refresh gated on health checks — new instances must pass before old ones terminate, making deploys genuinely zero-downtime.

Environments kept drifting

Manually-created dev and prod diverged constantly, producing "works in dev" failures.

Fixed with one set of Terraform modules and per-environment variable files over remote state — identical infrastructure stamped out per environment, changed only through pull requests.

Auto scaling thrashed

Inherited scaling policies scaled up and down repeatedly under perfectly steady load.

Fixed with re-tuned CloudWatch alarms — proper cooldowns and target-tracking policies based on real traffic patterns — stabilizing the fleet and cutting wasted instance-hours.
