Modular Terraform IaC Platform

Reusable, version-controlled Terraform modules provisioning EC2, VPC, RDS, S3, and DynamoDB across environments — paired with Ansible playbooks for configuration management. Infrastructure you can read, review, and reproduce.

The Mission

Hand-built cloud environments are snowflakes: undocumented, unrepeatable, and impossible to review. Every new environment meant days of console clicking and a fresh batch of subtle differences.

I codified the entire AWS footprint as modular Terraform — networking, compute, databases, and storage — so spinning up a complete, consistent environment became a pull request and a terraform apply. Ansible then handles in-instance configuration with community-backed roles.

What I Built

  • Reusable Terraform modules for EC2, VPC, RDS, S3, and DynamoDB with sane, overridable defaults.
  • Per-environment composition — dev and prod assembled from the same modules with different variable files.
  • Remote state with locking for safe, collaborative changes.
  • Ansible playbooks automating system configuration and provisioning via community roles.
  • Pipeline-driven applies — infrastructure changes flow through the same review-and-deploy discipline as application code.

Challenges & Solutions

Copy-pasted Terraform diverging per project

Early configurations were duplicated between projects, so fixes never propagated.

Solution: Extracted shared modules with versioned interfaces — one fix, every environment benefits on the next apply.

State conflicts between engineers

Local state files caused overwritten resources and "who changed this?" mysteries.

Solution: Remote backend with state locking and workspaces per environment, making concurrent work safe and auditable.

Provisioned servers still needed manual setup

Terraform created instances, but packages, users, and services were configured by hand.

Solution: Ansible playbooks run post-provision — idempotent configuration from inventory generated by Terraform outputs, closing the gap between "created" and "ready".
